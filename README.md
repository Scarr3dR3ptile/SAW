# SCRE · Steam Artwork Editor — Vercel-версия

Самодостаточная веб-версия для деплоя на **Vercel** без Docker и своего сервера.

## Файлы
```
index.html            весь клиентский функционал одним файлом
                      (UI, нарезка статики на Canvas, нарезка+сжатие анимации
                       через ffmpeg.wasm, макет профиля, 1:1-хром, каталог)
api/proxy.js          serverless: прокси Steam (обход CORS) — каталог, профиль,
                      ссылки, картинки, 1:1-страница профиля
api/steam-auth.js     serverless: вход Steam OpenID (login/return/me/logout)
ffmpeg/               ffmpeg.wasm (same-origin, чтобы Worker грузился без CORS):
  ffmpeg.js, 814.ffmpeg.js, esm/ffmpeg-core.js, esm/ffmpeg-core.wasm (~31 МБ)
package.json          Node ≥18, ESM
vercel.json           конфиг функций
```
> Папку `ffmpeg/` **обязательно** заливать вместе со всем — ffmpeg.wasm грузится
> с того же домена (иначе Worker блокируется браузером). COOP/COEP не нужны:
> используется single-thread ядро (ESM), поэтому 1:1-хром через iframe не ломается.

## Почему две функции на сервере
Из браузера напрямую **нельзя**:
* дёргать Steam API/страницы — Steam не отдаёт CORS → нужен прокси (`proxy.js`);
* завершить вход OpenID — нужна серверная проверка ответа (`steam-auth.js`).

Всё остальное (нарезка, сжатие, макет) работает **в браузере**.

## Деплой
**Вариант 1 — через сайт Vercel (проще):**
1. Залейте эту папку в репозиторий GitHub.
2. На vercel.com → *Add New → Project* → импортируйте репозиторий.
3. Deploy. Готово: `https://ваш-проект.vercel.app`.

**Вариант 2 — CLI:**
```bash
npm i -g vercel
cd vercel
vercel            # предпросмотр
vercel --prod     # продакшн
```

Домен Vercel всегда с HTTPS, поэтому **вход Steam работает сразу** — `return_to`
формируется от адреса деплоя автоматически.

## Важные нюансы (честно)
* **Сжатие GIF/видео идёт в браузере** через `ffmpeg.wasm` (single-thread ESM,
  без COOP/COEP), файлы ядра лежат в `ffmpeg/` (same-origin). Первая операция
  подгружает ядро (~31 МБ) и работает медленнее нативного; сжатие использует
  палитру/fps/разрешение (без gifsicle `lossy`), с синхронизацией fps между
  кусками.
* **Нарезка статичных** фонов — мгновенно, на Canvas.
* В режиме **1:1-хром** анимированный фон показывается статичным кадром
  (страница всё равно 1:1 по вёрстке); анимацию куски показывают в canvas-режиме.
* Профиль в canvas-режиме — точная геометрия витрины (пиксель-в-пиксель
  выравнивание кусков с фоном).

## Локальный запуск
```bash
npm i -g vercel
vercel dev        # поднимает и статику, и функции на localhost
```
