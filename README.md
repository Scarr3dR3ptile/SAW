# SCRE · Steam Artwork Editor — Vercel-версия

Самодостаточная веб-версия для деплоя на **Vercel** без Docker и своего сервера.

## Файлы
```
index.html            весь клиентский функционал одним файлом
                      (UI, нарезка статики на Canvas, нарезка+сжатие анимации
                       через ffmpeg.wasm, макет профиля, 1:1-хром, каталог)
api/proxy.js          serverless: прокси Steam (обход CORS) — каталог, профиль,
                      ссылки, картинки, 1:1-страница профиля
api/steam-auth.js     serverless: вход Steam OpenID (login/return/me/logout)
ffmpeg/               ffmpeg.wasm (same-origin, чтобы Worker грузился без CORS):
  ffmpeg.js, 814.ffmpeg.js, esm/ffmpeg-core.js, esm/ffmpeg-core.wasm (~31 МБ)
package.json          Node ≥18, ESM
vercel.json           конфиг функций
```
> Папку `ffmpeg/` **обязательно** заливать вместе со всем — ffmpeg.wasm грузится
> с того же домена (иначе Worker блокируется браузером). COOP/COEP не нужны:
> используется single-thread ядро (ESM), поэтому 1:1-хром через iframe не ломается.

## Почему две функции на сервере
Из браузера напрямую **нельзя**:
* дёргать Steam API/страницы — Steam не отдаёт CORS → нужен прокси (`proxy.js`);
* завершить вход OpenID — нужна серверная проверка ответа (`steam-auth.js`).

Всё остальное (нарезка, сжатие, макет) работает **в браузере**.

## Деплой
**Вариант 1 — через сайт Vercel (проще):**
1. Залейте эту папку в репозиторий GitHub.
2. На vercel.com → *Add New → Project* → импортируйте репозиторий.
3. Deploy. Готово: `https://ваш-проект.vercel.app`.

**Вариант 2 — CLI:**
```bash
npm i -g vercel
cd vercel
vercel            # предпросмотр
vercel --prod     # продакшн
```

Домен Vercel всегда с HTTPS, поэтому **вход Steam работает сразу** — `return_to`
формируется от адреса деплоя автоматически.

## Важные нюансы (честно)
* **Сжатие GIF/видео идёт в браузере** через `ffmpeg.wasm` (single-thread ESM,
  без COOP/COEP), файлы ядра лежат в `ffmpeg/` (same-origin). Первая операция
  подгружает ядро (~31 МБ) и работает медленнее нативного; сжатие использует
  палитру/fps/разрешение (без gifsicle `lossy`), с синхронизацией fps между
  кусками.
* **Нарезка статичных** фонов — мгновенно, на Canvas.
* **Профиль и превью нарезки — настоящая страница Steam** (в iframe, через
  `/api/proxy?action=profile-page`): вкладка «Профиль» показывает реальный профиль
  с вашим фоном; вкладка «Нарезка» — слева то же превью, но с нарезанными кусками,
  вставленными в настоящую витрину Steam (bg-режим — фон просвечивает сквозь окно
  витрины = бесшовно), справа настройки.
* Каталог: выбор анимированного фона автоматически подстраивает нарезку и
  открывает превью на вкладке «Нарезка».
* **Мобильная вёрстка**: адаптив включён (вкладки скроллятся, «Нарезка» в одну
  колонку, каталог 2-в-ряд, нет горизонтального переполнения).

## Вычисления на своём сервере (вместо ffmpeg.wasm в браузере)
По умолчанию нарезка/сжатие GIF идут в браузере (ffmpeg.wasm, ~31 МБ). Чтобы
перенести тяжёлое на **свой сервер** (реальный ffmpeg — быстрее, без 31 МБ у
клиента):
1. Поднимите FastAPI-бэкенд из основного проекта (`python -m scre_editor.web
   --https` или за Caddy) на публичном HTTPS-домене. Он уже умеет `/api/slice`
   и `/api/fit-gif` на настоящем ffmpeg, CORS включён.
2. В `index.html` вверху задайте адрес:
   ```js
   const COMPUTE_BACKEND="https://scre.ваш-домен.ру";
   ```
   Пусто = считать в браузере (запасной вариант).
3. На бэкенде сузьте CORS до своего домена Vercel:
   `SCRE_CORS_ORIGINS=https://ваш-проект.vercel.app` (env). По умолчанию `*`.

Тогда фронт (Vercel) шлёт фон/параметры на ваш сервер, тот режет/сжимает
настоящим ffmpeg и возвращает ZIP — «веб только принимает запросы».
Статичная нарезка остаётся мгновенной на Canvas.

**Отказоустойчивость (авто-фолбэк):** перед отправкой фронт проверяет живость
сервера лёгким `/api/geometry` (таймаут 4 с). Если сервер недоступен или не
ответил (или запрос оборвался по таймауту 120 с) — обработка **автоматически
переключается на браузер** (ffmpeg.wasm), пользователь всё равно получит ZIP.
В статусе видно, чем посчитано: «Готово (сервер)» или «Готово (браузер)».

## Реклама
В `index.html` два слота: `<div class="ad" id="adTop">` (после шапки) и
`<div class="ad" id="adBottom">` (внизу). Вставьте внутрь код своей сети
(Google AdSense, Adsterra и т.п.). Пустые слоты показывают плейсхолдер
«Место для рекламы» и исчезают, как только вставлен реальный код. Для AdSense
добавьте их скрипт в `<head>` и `<ins class="adsbygoogle" …>` внутрь слота.

```bash
npm i -g vercel
vercel dev        # поднимает и статику, и функции на localhost
```
